# nordocs C# bindings

The C# surface over the `nordocs` cdylib. See `../../../docs/ffi-packaging.md`
for how to build the native library per platform and wire it into a project.

## Files

- **`NordocsFfi.g.cs`** — generated by interoptopus. The flat P/Invoke
  `DllImport`s and `[StructLayout]` DTOs. **Do not edit**: regenerate with
  `cargo test -p nordocs-ffi generate_csharp_binding`. CI fails if it is stale.
- **`NordocsSession.cs`** — hand-written companion. Turns the flat surface into
  idiomatic C#: `byte[]`/`string` results, a thrown `NordocsException` instead of
  an out-parameter, and the opaque session handle as an `IDisposable`
  `NordocsSession` (so callers write `using var session = ...`, mirroring the
  reference's `using var compiler = ...`).
- **`consumer-test/`** — an optional, toolchain-gated round-trip that loads the
  built cdylib (see packaging doc). The Rust-side FFI smoke tests are
  authoritative.

## Reference parity and intentional divergences

The exported surface covers every operation of the reference `ITypstCompiler`,
`IPreviewRenderer`, and `IMarkdownToTypstConverter` interfaces. The Rust parity
test `ffi_surface_covers_reference_interfaces` (in `src/generate.rs`) enforces
this and lists the mapping. Divergences are deliberate and shape-only — the FFI
is a flat C ABI, so rich C# types are marshalled as UTF-8 / JSON:

| Reference operation | FFI export | Divergence |
| --- | --- | --- |
| `ITypstCompiler.CompileToPdf(source, Dictionary<string,string>?)` | `ndoc_compile_to_pdf(source, vars)` | `vars` is a JSON object string of `sys.inputs` |
| `ITypstCompiler.CompileFileToPdf(path)` | `ndoc_compile_file_to_pdf(path)` | none |
| `IMarkdownToTypstConverter.Convert(md)` | `ndoc_markdown_to_typst(md)` | none |
| `IPreviewRenderer.RenderComponentPreview(source, ComponentSchema, Dictionary, theme)` | `ndoc_render_component_preview(...)` | schema/inputs passed as JSON strings |
| `IPreviewRenderer.RenderDocumentPreview(DocumentState, theme, Dictionary, Dictionary)` | `ndoc_render_document_preview(...)` | state/maps passed as JSON strings |

In all cases the reference returns `byte[]`/`string` and throws on error; the FFI
returns a `ByteBuffer`/`FfiString` (pointer + length, freed once) plus an
out-parameter `FfiError`. The hand-written wrapper restores the
`byte[]`/`string`-and-throw shape, so swapping the implementation stays
mechanical.

The FFI additionally exposes capabilities beyond the three reference interfaces
(multi-format `Compile`, the authoring/validation/catalogue operations, and the
source-map session); these have no reference interface to mirror.
